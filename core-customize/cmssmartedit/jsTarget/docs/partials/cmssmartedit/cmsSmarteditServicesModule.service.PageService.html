PageService
service in module cmsSmarteditServicesModule

Description

The pageServiceModule provides services for dealing with CMS page objects

Methods

  • buildUriContextForCurrentPage()

    Returns the uriContext populated with the siteId, catalogId and catalogVersion taken from $routeParams and fallback to the currentExperience Note: From the page list, $routeParams are defined. From the storefront, $routeParams are undefined.

    Returns
    Promise<IUriContext>

    promise resolve to the uri context

  • forcePageApprovalStatus()

    This method will forcefully update the page approval status (as long as the current user has the right permissions) of the page loaded in the current context to the given status.

    Returns
    Promise<ICMSPage>

    If request is successful, it returns a promise that resolves with the updated CMS Item object. If the request fails, it resolves with errors from the backend.
