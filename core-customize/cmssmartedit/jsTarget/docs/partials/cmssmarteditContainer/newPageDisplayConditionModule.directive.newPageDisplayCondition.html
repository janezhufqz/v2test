newPageDisplayCondition
directive in module newPageDisplayConditionModule

Description

Component for selecting the page condition that can be applied to a new page. The component takes a page type and some URI params that it needs to load the necessary information, and outputs a display condition result. See below

Usage

as element:
<new-page-display-condition
       page-type-code="{< String}"
       uri-context="{< Object}"
       [result-fn="{< Function }"]
       [initial-primary-page-selected="{< String }"]
       [initial-condition-selected="{< String }"]
       [target-catalog-version="{< String }"]>
</new-page-display-condition>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
pageTypeCode< String

The page typeCode of a potential new page

uriContext< Object

The uri context containing site/catalog information. This is necessary for the component to determine which display conditions can be applied.

PropertyTypeDetails
siteUIDString

The site ID for the new page

catalogIdString

The catalog ID for the new page

catalogVersionString

The catalog version for the new page

resultFn
(optional)
< Function

An optional output function binding. Every time there is a change to the output, or resulting display condition, this function (if it exists) will get executed with a newPageDisplayConditionResult as the single parameter.

initialPrimaryPageSelected
(optional)
< String

A page label if primary page associated to the variation must be selected by default.

initialConditionSelected
(optional)
< String

An optional string to provide if any display condition must be selected by default.

targetCatalogVersion
(optional)
< String

An optional string to provide if the target catalog version is selected
