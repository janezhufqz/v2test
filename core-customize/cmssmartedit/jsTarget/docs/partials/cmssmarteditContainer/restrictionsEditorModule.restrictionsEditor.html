restrictionsEditor
service in module restrictionsEditorModule

Description

The purpose of this directive is to allow the user to manage the restrictions for a given item. The restrictionsEditor has an editable and non-editable mode. It uses the restrictionsTable to display the list of restrictions and it uses the restrictionsPicker to add or remove the restrictions.

Usage

as element:
<restrictions-editor
       item="{= Object}"
       editable="{= Boolean}"
       [restrictions="{< Array}"]
       [reset-fn="{= Function}"]
       [cancel-fn="{= Function}"]
       [is-dirty-fn="{= Function}"]
       [on-restrictions-changed="{& Expression}"]
       get-restriction-types="{& Expression}"
       [get-supported-restriction-types="{& Expression}"]>
</restrictions-editor>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
item= Object

The object for the item you want to manage restrictions.

PropertyTypeDetails
onlyOneRestrictionMustApplyBoolean

The restriction criteria for the item.

uuidString

The uuid of the item. Required if not passing initialRestrictions. Used to fetch and update restrictions for the item.

uriContextObject

the uriContext

editable= Boolean

Boolean to determine whether the editor is enabled.

restrictions
(optional)
< Array

An array of initial restriction objects to be loaded in the restrictions editor. If restrictions is not provided, it is used else it is assumed that there are no restrictions.

resetFn
(optional)
= Function

Function that returns true. This function is defined in the restrictionsEditor controller and exists only to provide an external callback.

cancelFn
(optional)
= Function

Function that returns a promise. This function is defined in the restrictionsEditor controller and exists only to provide an external callback.

isDirtyFn
(optional)
= Function

Function that returns a boolean. This function is defined in the restrictionsEditor controller and exists only to provide an external callback.

onRestrictionsChanged
(optional)
& Expression

Function that passes '$onlyOneRestrictionMustApply' boolean and an array of '$restrictions' as arguments. The invoker can bind this to a custom function to fetch these values and perform other operations.

getRestrictionTypes& Expression

Function that return list of restriction types. The invoker can bind this to a custom function to fetch a list of restriction types.

getSupportedRestrictionTypes
(optional)
& Expression

Function that returns an arry of supported restriction types. The invoker can bind this to a custom function to fetch these values and perform other operations. If not provide, all types are assumed to be supported.
